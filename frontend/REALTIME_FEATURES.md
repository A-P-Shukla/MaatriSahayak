# Real-time Features Documentation

## WebSocket Integration

### Overview
The frontend now includes real-time updates via WebSocket connections for:
- Emergency alerts
- Ambulance location tracking
- Pregnancy status updates
- Dashboard statistics

### Configuration

Add WebSocket URL to `.env`:
```env
VITE_WS_URL=wss://73qjqd2j7c.execute-api.ap-south-1.amazonaws.com/dev
```

### Usage

The `useWebSocket` hook automatically:
- Connects to WebSocket server on mount
- Authenticates using stored access token
- Invalidates React Query cache when updates arrive
- Reconnects automatically (up to 5 attempts)
- Disconnects on unmount

```typescript
import { useWebSocket } from '../hooks/useWebSocket';

const { isConnected, lastMessage, sendMessage } = useWebSocket({
  enabled: true,
  reconnectInterval: 5000,
  maxReconnectAttempts: 5,
});
```

### Message Types

WebSocket messages follow this format:
```typescript
{
  type: 'emergency_update' | 'ambulance_location' | 'pregnancy_update' | 'stats_update',
  data: any
}
```

## Map Integration

### Technology
- **Library**: React Leaflet (wrapper for Leaflet.js)
- **Tile Provider**: OpenStreetMap (free, no API key required)
- **Features**: 
  - Real-time ambulance markers
  - Custom status-based icons (green=available, orange=dispatched, red=busy)
  - Interactive popups with ambulance details
  - Auto-centering on ambulance locations
  - Fullscreen mode

### Alternative Map Providers

To use Google Maps or Mapbox instead:

#### Google Maps
1. Install: `npm install @react-google-maps/api`
2. Get API key from Google Cloud Console
3. Replace `AmbulanceMap.tsx` with Google Maps component

#### Mapbox
1. Install: `npm install react-map-gl mapbox-gl`
2. Get access token from Mapbox
3. Add to `.env`: `VITE_MAPBOX_TOKEN=your_token`
4. Replace tile layer in `AmbulanceMap.tsx`:

```typescript
<TileLayer
  attribution='&copy; <a href="https://www.mapbox.com/">Mapbox</a>'
  url="https://api.mapbox.com/styles/v1/mapbox/streets-v11/tiles/{z}/{x}/{y}?access_token={accessToken}"
  accessToken={import.meta.env.VITE_MAPBOX_TOKEN}
/>
```

### Current Implementation

The map uses OpenStreetMap tiles (no API key needed):
```typescript
<TileLayer
  attribution='&copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a> contributors'
  url="https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png"
/>
```

## Export Reports

### Functionality
Analytics page now supports exporting reports in two formats:
- **PDF**: Generated by backend `/analytics/export` endpoint
- **CSV**: Fallback generated client-side if backend unavailable

### Usage
Click "Export Report" button on Analytics page. The system will:
1. Attempt to fetch PDF from backend
2. If backend fails, generate CSV locally
3. Auto-download file with date range in filename

## Backend Requirements

### WebSocket Lambda
Create API Gateway WebSocket API with routes:
- `$connect`: Authenticate connection
- `$disconnect`: Clean up connection
- `$default`: Handle messages

### Export Lambda
Endpoint: `POST /analytics/export`
Request body:
```json
{
  "start_date": "2024-01-01",
  "end_date": "2024-01-31",
  "format": "pdf"
}
```
Response: Binary PDF file

## Performance

### WebSocket
- Automatic reconnection on disconnect
- Message batching to prevent flooding
- Query invalidation debounced

### Map
- Markers update only when data changes
- Lazy loading of map tiles
- Efficient re-rendering with React.memo

### Export
- Streaming for large reports
- Client-side CSV generation as fallback
- Progress indication for large exports
